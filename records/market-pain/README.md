# Market Pain Records

This directory stores market-pain research records that can feed the Revenue Pipeline.

Status: the standalone recurring `market-pain-radar` automation is inactive. Future recurring runs should wait until the project can turn records into judgment, one selected bet, and execution artifacts.

Default record filename:

```text
YYYY-MM-DD-HHMM.md
```

Each record should start from public community pain signals, then map the strongest signals to GitHub repositories or other open-source tools that could help a solo developer prototype a small 2C product.

ERP-related applications and heavy enterprise operations software are excluded from default records.
